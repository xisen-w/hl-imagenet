# logs/phase2/diagnostics

<!-- RCC-MINI-README:START -->

## Purpose

Generated Phase 2 diagnostic reports for attractor/victim analysis over existing evaluation logs.

## S - Formal specification

This folder stores derived diagnostic artifacts generated by scripts/run_phase2_diagnostics.py. Diagnostics analyze existing logs/phase2/eval_phase2*.json files and do not change classifier behavior.

## H - Hooks and integration edges

Consumes Phase 2 evaluation JSON logs from logs/phase2/. Generated reports inform future Phase 2.3 classifier or scoring work but must not be treated as classifier improvements by themselves.

## A - Artifacts

- latest_phase2_diagnostic.json
- latest_phase2_diagnostic.md
- future timestamped diagnostic reports if added

## T - Theory or method basis

The diagnostic lens treats classifier failures as a confusion-field geometry: predicted classes can become false-positive attractors, true classes can become victims, and repeated true-to-predicted errors form confusion gravity wells.

## I - Invariants

- Diagnostics do not prove classifier correctness.
- Diagnostics do not change classifier behavior.
- Diagnostics do not imply RCC improved classifier accuracy.
- Diagnostics should guide inspection before scorer or signature changes.

## E - Example

Run from the repository root:

    python scripts/run_phase2_diagnostics.py
