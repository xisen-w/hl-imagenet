# logs/phase2/attribution

## Purpose

Generated Phase 2 sample-level attribution artifacts for inspecting individual validation predictions, collapse paths, activated features, proof traces, and optional baseline agreement.

## S - Formal specification

This folder stores artifacts generated by `scripts/run_phase2_attribution.py`. The attribution layer reads the local Phase 2 image split, runs the current HL symbolic classifier, and emits per-sample JSON/CSV/Markdown reports without changing classifier behavior.

## H - Hooks and integration edges

- `hlinet/eval/attribution.py` runs sample-level attribution.
- `scripts/run_phase2_attribution.py` is the human-facing runner.
- `hlinet/classifier/predict.py` supplies prediction labels, alternatives, proof traces, and feature activations.
- `hlinet/eval/baselines.py` may supply optional baseline agreement.
- Outputs inform future Phase 2.5 attractor-balanced scoring.

## A - Artifacts

- `latest_phase2_attribution.json`
- `latest_phase2_attribution.csv`
- `latest_phase2_attribution.md`

## T - Theory or method basis

Aggregate diagnostics show where the classifier fails. Sample-level attribution shows why individual images collapse into false-positive attractors or survive as correct predictions.

## I - Invariants

- Attribution tooling does not change classifier behavior.
- Validation attribution is not final test evidence.
- Proof traces explain model behavior; they do not prove correctness.
- Do not tune classifier behavior from this artifact without a new architecture lock and benchmark rerun.
- Do not commit local Tiny ImageNet image data.

## E - Example

Smoke run:

    python scripts/run_phase2_attribution.py --data-root ".\data\phase2" --split val --eval-max-per-class 20 --train-max-per-class 20

Full validation run:

    python scripts/run_phase2_attribution.py --data-root ".\data\phase2" --split val
