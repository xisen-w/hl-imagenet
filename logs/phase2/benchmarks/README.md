## Purpose

Generated Phase 2 benchmark artifacts comparing the current HL symbolic classifier against simple non-neural baselines on the same declared split.

## S - Formal specification

This folder stores benchmark outputs generated by scripts/run_phase2_benchmarks.py. Benchmark reports compare transparent baselines against the current HL classifier without modifying classifier behavior.

## H - Hooks and integration edges

- hlinet/eval/baselines.py defines random, majority, color-centroid, image-statistics centroid, and handcrafted kNN baselines.
- hlinet/eval/benchmark.py fits baselines on the train split and evaluates on the requested split.
- scripts/run_phase2_benchmarks.py is the human-facing runner.
- Benchmark artifacts inform future Phase 2.4 attribution or Phase 2.5 classifier work.

## A - Artifacts

- latest_phase2_benchmark.json
- latest_phase2_benchmark.md
- future timestamped benchmark reports if added

## T - Theory or method basis

Benchmark discipline asks whether the symbolic classifier beats transparent non-neural baselines on the same split before making stronger performance claims or changing the classifier.

## I - Invariants

- Benchmark tooling does not change classifier behavior.
- Baselines fit only on train.
- Validation is not final test.
- Do not claim classifier improvement from benchmark tooling.
- Do not claim symbolic methods beat neural systems from these baselines.

## E - Example

Run from the repository root:

    python scripts/run_phase2_benchmarks.py --split val

For a faster smoke run:

    python scripts/run_phase2_benchmarks.py --split val --train-max-per-class 20 --eval-max-per-class 20
