## Purpose

Human-readable and machine-readable RCC / AEFL process metrics for HL-ImageNet.

## S - Formal specification

This folder stores generated repository-process metrics. These metrics summarize README/RCC coverage, command coverage, boundary coverage, evidence artifact coverage, mini README coverage, rejected-delta learning, and runtime-integrity status after the regression guard was introduced.

## H - Hooks and integration edges

- Generated by `scripts/metrics/generate_rcc_process_dashboard.py`.
- Referenced by the human README section `RCC / AEFL Context Layer: Repository Process Metrics`.
- Uses README, logs, rejected-delta ledgers, and repository file presence as inputs.
- Links to charts in `docs/plots/`.

## A - Artifacts

- `rcc_process_metrics.md`
- `rcc_process_metrics.json`

## T - Theory or method basis

RCC turns repository context into durable navigation and governance surfaces. The metrics here measure repository-process quality: context location, claim-boundary safety, evidence-chain completeness, drift resistance, failure learning, and controlled evolution.

## I - Invariants

- These metrics do not measure classifier correctness.
- These metrics do not claim ImageNet performance.
- These metrics do not imply RCC changed classifier runtime behavior.
- Coverage is not proof.
- Charts are process-observability artifacts, not benchmark results.

## E - Example

Regenerate from the repository root:

    python scripts/metrics/generate_rcc_process_dashboard.py
