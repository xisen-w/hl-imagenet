\begin{abstract}
HL-ATTR v1.0 defines a sample-level attribution layer for HL-ImageNet. It reads
a declared Phase 2 split, runs the current symbolic classifier without changing
runtime behavior, and emits per-sample JSON, CSV, and Markdown artifacts
containing true labels, predictions, top-3 labels, margins, proof traces,
activated features, collapse paths, and optional baseline agreement. The layer
is a diagnostic/evidence instrument, not a classifier improvement.
\end{abstract}

\section{Core Invariant}

\[
\boxed{
\text{Aggregate diagnostics show where the system fails; sample attribution}
\atop
\text{shows why individual images collapse.}
}
\]

\[
Sample
\rightarrow Prediction
\rightarrow Top3
\rightarrow Margin
\rightarrow Features
\rightarrow Proof
\rightarrow CollapsePath
\rightarrow AttributionArtifact.
\]

\section{Artifact Contract}

Each sample row must contain true label, predicted label, top-3 labels, score
margin, outcome class, collapse path, activated features, and proof trace.

\section{Outcome Classes}

\[
Outcome \in \{correct,\ top3\_rescue,\ miss\}.
\]

\section{Non-Claim Lock}

This layer does not prove correctness, does not change classifier behavior, and
does not claim accuracy improvement. It exists to inspect failures before any
future scoring or signature changes.

\section{Next Step}

After HL-ATTR v1.0 emits stable sample-level traces, Phase 2.5 may introduce
attractor-balanced scoring only if changes are benchmarked against the same
baselines and validation/test boundaries remain preserved.
