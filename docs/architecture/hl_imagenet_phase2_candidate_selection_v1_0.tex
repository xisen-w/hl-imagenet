\begin{abstract}
HL-CAND v1.0 defines a candidate-selection layer for HL-ImageNet. It reads
diagnostic, benchmark, and sample-level attribution artifacts, then emits a
ranked candidate plan for future controlled classifier changes. The layer does
not modify classifier runtime behavior. Its purpose is to prevent blind feature
edits by identifying baseline-right/HL-wrong rows, HL-right/baseline-wrong rows,
overactive features, victim rescue targets, attractor suppression targets, and
regression risks before Phase 2.6.
\end{abstract}

\section{Core Invariant}

\[
\boxed{
\text{Do not change classifier behavior until the evidence stack selects}
\atop
\text{which failure mode deserves the first controlled delta.}
}
\]

\[
Diagnostics + Benchmarks + Attribution
\rightarrow CandidatePlan
\rightarrow ControlledDelta
\rightarrow RerunEvidence.
\]

\[
Candidate \in

BaselineRightHLWrong,

RegressionGuard

\section{Non-Claim Lock}

This layer ranks possible future interventions. It does not prove correctness,
does not improve accuracy by itself, and does not authorize validation-tuned
claims.

\section{Next Step}

Phase 2.6 may implement one controlled classifier delta only after a selected
candidate is tied to attribution rows and all diagnostic/benchmark/attribution
reruns are required.
