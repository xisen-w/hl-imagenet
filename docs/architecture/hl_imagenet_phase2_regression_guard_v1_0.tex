\begin{abstract}
HL-RG v1.0 defines a regression guard layer for HL-ImageNet. It reads benchmark,
attribution, and candidate-selection artifacts, locks the current evidence state
as a baseline, and emits pass/warning/fail guard rules for future classifier
changes. The layer does not modify classifier runtime behavior. Its purpose is
to protect benchmark discipline, HL-unique wins, victim-class recall, and
attractor-suppression goals before Phase 2.6B makes one controlled classifier
delta.
\end{abstract}

\section{Non-Claim Lock}

This layer locks regression expectations. It does not prove correctness, does
not improve accuracy by itself, and does not authorize validation-tuned claims.

Phase 2.6B may implement one controlled classifier delta only after this
